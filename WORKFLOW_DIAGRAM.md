## Key Points

1. **No changes to target repo**: Django repository is never modified
2. **Runs in YOUR repo**: All actions happen in v-mcp-2
3. **Clones are temporary**: Target repo cloned to `/tmp`, deleted after scan
4. **Results stored centrally**: All findings in `results/` in your repo
5. **Parallel execution**: Multiple scanners run at the same time
6. **Auto-committed**: Results pushed back to your repository
